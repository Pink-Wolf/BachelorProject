CompWolf::new_shader_field

In project Graphics.Core

Defined in header <shaders>

Member of

Inherits from: nothing.
template <typename... TypesAndOffsets> struct new_shader_field;

Implements everything a specialization of shader_field_info needs, which means specializations can just inherit from this instead of manually implementing the fields.

Warning

Template Parameters

NameDescription
TypesAndOffsets Must be some type_value_pairs, specifying the target type's various data members. The pairs should specifically specify their type and offset (offset can be calculated with offsetof). Only fields that need to be visible by the shader needs to be specified. All specified types must be ShaderFields themself. If any of this is not fulfilled, then a compile-time error is issued.

Members

NameDescription
(constructors)

primitivesThe primitive types making up the class/struct.
primitive_offsetsThe byte-position of the primitives in the class/struct.

Example

The following code defines a struct "vertex", and then specializes shader_field_info so this struct can be passed to shaders in the future. This specialization is defined using new_shader_field, where each field's type and position is listed in a type_value_pair.

#include <dimensions> // float2
#include <shaders> // shader_field_info, new_shader_field
#include <utility> // offsetof

struct vertex
{
    float2 position;
    float2 uv;
};
template<> struct shader_field_info<vertex> : public new_shader_field<
    type_value_pair<float2, offsetof(vertex, position)>,
    type_value_pair<float2, offsetof(vertex, uv)>
> {};
